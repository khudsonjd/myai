AI-Assisted Project Management

A repeatable, file-based system for working with AI across sessions, tools, and environments — without losing context.

Template & Process Guide
The Problem

AI forgets everything when the session ends

Every new chat starts from zero. Complex projects require deep context — your goals, decisions, history, and current state. Rebuilding that context every session is slow, error-prone, and unsustainable.

🔁

Stateless by default

No memory of previous sessions, decisions, or progress.

📋

Context gets lost

Important details live only in chat history — which gets cut or discarded.

🔧

Tool-dependent

Switching AI tools means starting from scratch — no portability.

The Solution

External files as persistent memory

Instead of relying on chat history, we store everything the AI needs in plain markdown files. The AI reads these at the start of every session and is immediately oriented — regardless of what tool you're using.

The AI doesn't need to remember anything between sessions. The files are the memory. The AI is the engine that reads, updates, and acts on them.

📁

Plain markdown

Works with any AI, any editor, any OS. No vendor lock-in.

♻️

Repeatable

Same structure works for any project — home, work, health, anything.

What It Is

A simple, intentional file structure

Four types of files, each with a clear purpose.

workspace/
  ├── SESSION.md  ← load at the start of every session
  ├── my-project/
  │   ├── project_plan.md  ← stable reference data
  │   └── status.md  ← current state, priorities, next steps
  ├── _template/  ← blank files to copy for new projects
  └── _meta/  ← process docs, decisions, improvement log
Why It's Structured This Way

Stable reference vs. active state

Every project uses two files — not one. This is intentional.

project_plan.md — what doesn't change

  • Goals and scope
  • Key facts and constraints
  • Methods and decisions already made
  • Long-horizon plans

status.md — what changes constantly

  • Current priorities
  • Deadlines and target dates
  • What's in progress or blocked
  • Supply needs, next actions

The AI reads status.md first for orientation, then reaches into project_plan.md only when deeper context is needed. Keeps sessions fast and focused.

How It Works

What happens at the start of every session

1

You start a session

Open your AI tool and point it at your workspace, or paste SESSION.md into the chat.

2

AI reads SESSION.md

Immediately oriented — knows who you are, what projects exist, and current focus.

3

AI reads status.md

Picks up current state — priorities, deadlines, what was last happening.

4

You work — AI keeps files current

As the session progresses, the AI updates status.md so the next session starts clean.

The Working Relationship

You direct. AI executes.

This system works best when roles are clearly defined — and that clarity is built into the structure itself.

You — the director

  • Set goals and priorities
  • Make key decisions
  • Provide context and corrections
  • Confirm before big changes

AI — the assistant

  • Research and analysis
  • Write and update files
  • Surface next steps and blockers
  • Document decisions and process

The AI asks clarifying questions before creating new files or making structural changes. It suggests — you decide.

Getting Started

Minimum needed to begin

You don't need to fill in everything upfront. The AI helps build context through conversation.

1

Copy the template folder

Rename it to your project name. Two files inside: SESSION.md and project_plan.md.

2

Fill in one line

Add your project name and a one-sentence description. That's all that's required to start.

3

Hand it to your AI

Share the file. The AI will ask questions, suggest structure, and build the rest with you.

📝

🔀
